# LLM-Hallucination-Detection
Создание классификатора для распознавания галлюцинаций ллм на базе GigaChat 

Проект для создания классификатора, который определяет, содержит ли ответ языковой модели галлюцинацию.  
В качестве базовой модели используется GigaChat, а классификация строится на признаках, извлечённых из внутренних состояний модели и статистик уверенности по токенам.

## Описание проекта

Проект включает два основных этапа:

1. **Подготовка русскоязычного датасета**
   - загрузка датасета FactCHD;
   - разбиение данных на части;
   - перевод вопросов и ответов на русский язык;
   - сохранение результата в CSV.

2. **Обучение и тестирование классификатора**
   - загрузка модели GigaChat;
   - извлечение признаков из hidden states выбранных слоёв;
   - расчёт признаков неопределённости;
   - обучение классификатора на базе XGBoost;
   - оценка качества по метрике PR-AUC.

## Подход к детекции галлюцинаций

Для каждого примера модель получает:

- пользовательский запрос;
- ответ модели.

Затем выполняется один forward pass через GigaChat.  
Во время прохода через модель собираются внутренние состояния выбранных слоёв.

Используются следующие группы признаков:

### 1. Uncertainty features

Признаки неопределённости считаются по logits модели на токенах ответа:

- средний log-probability;
- минимальный и максимальный log-probability;
- стандартное отклонение log-probability;
- средняя энтропия;
- максимальная энтропия;
- perplexity-like score;
- длина ответа в токенах;
- средняя вероятность top-1;
- суммарная вероятность top-5.

### 2. Internal features

Извлекаются скрытые состояния нескольких слоёв модели:

```python
PROBE_LAYERS = [0, 5, 10, 15, 20, 25]
```

Для выбранных слоёв сохраняются probe-векторы, которые затем используются как признаки для классификатора.

В рамках экспериментов был построен график точности классификации при использовании `probe_vec`, извлечённых с разных слоёв модели.

Такой анализ позволяет оценить, на каких уровнях GigaChat содержится наиболее полезная информация для детекции галлюцинаций.  

График точности по слоям помогает:

- выбрать наиболее информативные слои для `PROBE_LAYERS`;
- сравнить вклад разных уровней модели в задачу классификации;
- уменьшить размерность признакового пространства, оставив только полезные слои;
- повысить интерпретируемость baseline-подхода.

### 3. Классификатор

В качестве классификатора используется `XGBoost`.

Перед обучением признаки проходят через:

- `PCA`;
- `StandardScaler`;
- `XGBClassifier`.

## Инференс

Для инференса используется класс `GuardianOfTruth`.

Пример логики работы:

```python
guard = GuardianOfTruth(model, tokenizer, clf)

result = guard.score(
    prompt="Вопрос пользователя",
    answer="Ответ модели"
)

print(result.is_hallucination)
print(result.is_hallucination_proba)
```

Результат содержит:

| Поле | Описание |
|---|---|
| `is_hallucination` | бинарный результат классификации |
| `is_hallucination_proba` | вероятность галлюцинации |
| `t_model_sec` | время forward pass модели |
| `t_overhead_sec` | время работы классификатора и обработки признаков |
| `t_total_sec` | общее время обработки |

## Метрики

Для оценки используется метрика:

```text
PR-AUC
```

Метрика считается отдельно для train и test split.

Также выводится среднее время обработки одного примера:

- время forward pass модели;
- overhead на извлечение признаков и классификацию;
- p99 overhead.
